Learn CLI

pleeease compile

The first command is compile

For example, compile "foo.css" to "bar.css":

$ pleeease compile foo.css to bar.css

It's just easy as that!

You can compile more files at the same time. Here "foo.css" and "bar.css" will be concatenated to "baz.css":

$ pleeease compile foo.css bar.css to baz.css

You can also compile an entire folder, and copy file in another destination (if folders doesn't exist, they will be created):

$ pleeease compile css/ to public/css/app.min.css

pleeease watch

Last command but not least: watch. It's the same as compile, except that it activates live compilation. When a modification is detected, files will be recompiled automatically.

$ pleeease watch foo.css to bar.css

defaults

Pleeease is configured as this:

  • input is "*.css"
  • output is "app.min.css"

It means that you are not forced to use the long form syntax. For example:

$ pleeease compile

is the same as:

$ pleeease compile *.css to app.min.css

In the same way:

$ pleeease compile styles/foo.css

compile "styles.css/foo.css" to "app.min.css"

You can override default value with a configuration file.

Configuration file

Pleeease options can be set in a .pleeeaserc file (JSON-like), for example:

{
  "input": ["foo.css", "foo2.css"],
  "output": "bar.css"
}
  • input is an array of files (default ["*.css"])
  • output is the path to the compiled file (default app.min.css)

You can now compile easily:

$ pleeease compile

You can also overrides all postprocess options. For example, if you want to enable autoprefixer and disable minifier:

{
  "input": ["foo.css", "foo2.css"],
  "output": "bar.css",
  "autoprefixer": ["last 4 versions"],
  "minifier": false
}
